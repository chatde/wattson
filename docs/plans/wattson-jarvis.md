**Goal:** Turn Wattson (Samsung Galaxy Note 9, Termux, Ollama) into a Jarvis/TARS-class autonomous agent that speaks, listens, sees, learns, and self-upgrades — becoming smarter every day without human intervention.

**Architecture:** Plugin-based brain loop (watson-core.js) running on-device. Each capability is a plugin in ~/watson-plugins/. New models, new capabilities, and new knowledge all flow in autonomously. The phone becomes a self-improving agent that Dad can talk to like a real AI assistant.

**Tech Stack:** Node.js (Termux), Ollama HTTP API, termux-tts-speak (TTS), termux-microphone-record + termux-speech-to-text (STT), Android shell via ADB, Yahoo Finance / CoinGecko (markets), Tesseract (OCR), Termux:API (sensors/camera/mic)

**Deployment:** Changes live at `/Volumes/AI-Models/wattson/` on Mac. Push via ADB to `/data/data/com.termux/files/home/`. Plugin hot-reload is automatic within ~1 second.

### Task 1.4: Wake word daemon (watson-voice-daemon.js)

Separate process that continuously listens for "wattson", records the command, queries the brain, and speaks the answer.

- [x] **Step 1: Check termux-microphone-record availability**

Both present: `termux-microphone-record`, `termux-speech-to-text`

- [x] **Step 2: Test microphone recording**

Flag is `-f` not `-o`. Records AAC format. Command starts recording and returns immediately.
Need to `sleep(duration)` then send `-q` stop command before reading file.

- [x] **Step 3: Speech-to-text decision**

`termux-speech-to-text` opens Android dialog — NOT usable in background.
Solution: Use **Groq Whisper API** (key already in dashboard). Faster, background-compatible, free tier.
STT format: POST multipart/form-data to `api.groq.com/openai/v1/audio/transcriptions`, model `whisper-large-v3-turbo`.

- [x] **Step 4: Create `watson-voice-daemon.js`**

Core loop implemented:
1. Record CHUNK_SECS (3) of audio via execFile termux-microphone-record
2. Transcribe via Groq Whisper (HTTP POST, no shell)
3. Check transcript.toLowerCase().includes('wattson')
4. If yes: speak "Yes?", record 5s command audio, transcribe
5. Send command to Ollama wattson:chat via HTTP POST body
6. Speak response via TTS

Security: execFile with arg arrays throughout. Ollama via HTTP POST. Groq via HTTPS.

- [x] **Step 5: Deploy daemon**

Deployed to `/data/data/com.termux/files/home/watson-voice-daemon.js`
Log: `/sdcard/wattson-voice.log` — confirmed: `[daemon] Wattson voice daemon started. Listening for wake word...`

- [x] **Step 6: Add to start-watson.sh**

Added step 5.5 to `/Volumes/AI-Models/wattson/start-watson.sh` — starts voice daemon alongside core/dashboard.

## Deployment Quick Reference

### Deploy any plugin
```bash

PLUGIN=watson-plugins/PLUGINNAME.plugin.js
adb -s 192.168.4.32:5555 push $PLUGIN /sdcard/$(basename $PLUGIN)
adb -s 192.168.4.32:5555 shell "run-as com.termux sh -c 'cp /sdcard/$(basename $PLUGIN) /data/data/com.termux/files/home/watson-plugins/$(basename $PLUGIN) && echo OK'"
# Hot-reload fires in ~1s — check: adb shell "tail -3 /sdcard/watson-thoughts.log"
```

### Deploy watson-core.js
```bash
adb -s 192.168.4.32:5555 push watson-core.js /sdcard/watson-core.js
adb -s 192.168.4.32:5555 shell "run-as com.termux sh -c 'cp /sdcard/watson-core.js ~/watson-core.js'"
# Restart: SIGTERM only
adb -s 192.168.4.32:5555 shell "kill $(ps -A 2>/dev/null | grep watson-core | awk '{print $1}' | head -1)"
```
